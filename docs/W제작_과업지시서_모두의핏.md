# < S/W 제작 > 과업지시서 - 모두의핏 체육시설 통합 플랫폼

## 1. 과업 내용

### ❍ 반응형 웹 애플리케이션 방식으로 시스템 구축
- **크로스 플랫폼 지원**: 데스크톱, 태블릿, 모바일 기반 모두 지원
- **PWA(Progressive Web App) 기술 적용**: 앱과 유사한 사용자 경험 제공
- **제작된 웹앱의 도메인 등록 및 QR코드 생성**: `https://all4fit.co.kr` 도메인 등록 및 접속용 QR코드 제공

### ❍ 체육시설 통합 플랫폼 주요 서비스를 웹 애플리케이션으로 구축
- **시설 검색 및 필터링**: 지역, 종목, 시설 유형별 고급 검색 기능
- **시설 상세 정보 제공**: 위치, 연락처, 이용시간, 요금, 시설 사진 등 종합 정보
- **바우처 정보 관리**: 스포츠강좌이용권 신청 및 안내 서비스
- **사용자 맞춤 서비스**: 즐겨찾기, 검색 기록, 개인화된 추천 기능

### ❍ 실시간 알림서비스 기능 구현 및 모바일 홍보서비스 개발
- **웹 푸시 알림**: 브라우저 기반 푸시 알림 서비스 (Service Worker 활용)
- **이벤트 알림**: 시설별 이벤트, 공지사항 실시간 알림
- **관리자 알림 시스템**: 관리자 대시보드를 통한 푸시 메시지 전송 기능
- **SNS 연동**: 체육시설 정보 공유 및 홍보 기능 (향후 확장)

### ❍ 웹 애플리케이션 운영 및 컨텐츠 등록 등 관리시스템 구축
- **관리자 대시보드**: 직관적인 웹 기반 관리 인터페이스
- **시설 정보 CRUD**: 시설 등록, 수정, 삭제, 조회 기능
- **컨텐츠 관리**: 이미지, 동영상, 문서 등 멀티미디어 컨텐츠 관리
- **사용자 관리**: 회원 가입, 로그인, 권한 관리 시스템
- **통계 및 분석**: 접속 통계, 사용자 행동 분석, 성능 모니터링

### ❍ 보안 및 개인정보보호 준수
- **사용자 및 관리자 권한 설정을 통한 접근 제한 기능 구현**
  - 역할 기반 접근 제어(RBAC) 시스템
  - 관리자, 일반사용자, 게스트 권한 분리
  - 세션 관리 및 자동 로그아웃 기능
- **개인정보보호 강화를 위한 네트워크 구간 보안통신(HTTPS) 적용**
  - SSL/TLS 1.3 암호화 통신
  - HSTS(HTTP Strict Transport Security) 적용
  - 보안 헤더 설정 (CSP, X-Frame-Options 등)
- **Secure Coding 가이드라인을 준수하여 소스코드 작성**
  - OWASP Top 10 보안 취약점 대응
  - XSS, CSRF, SQL Injection 방지
  - 입력값 검증 및 출력 인코딩

### ❍ S/W 도입
- **보안통신을 위한 SSL(Secure Socket Layer) 인증서**
  - Let's Encrypt 무료 SSL 인증서 적용
  - 자동 갱신 시스템 구축
  - Wildcard 인증서로 서브도메인 지원
- **입력 값 노출 및 변조 방지를 위한 클라이언트 사이드 보안**
  - Content Security Policy(CSP) 적용
  - 입력 필드 마스킹 및 검증
  - 클라이언트 사이드 데이터 암호화
- **인증 및 보안을 위한 PKI 솔루션**
  - JWT(JSON Web Token) 기반 인증
  - 비밀번호 해싱 (bcrypt)
  - 세션 관리 및 토큰 갱신

## 2. 세부 사항

## 1. 웹 접근성 및 웹 표준 준수

### 가. 웹 접근성 지침 준수
모든 페이지는 '한국형 웹 콘텐츠 접근성 지침 2.0'을 준수하여 구축한다.
- **인식의 용이성**: 대체 텍스트, 색상 대비, 텍스트 크기 조절 기능 제공
- **운용의 용이성**: 키보드 접근성, 충분한 링크 영역, 일관된 네비게이션
- **이해의 용이성**: 명확한 제목 구조, 읽기 쉬운 콘텐츠, 오류 메시지 제공
- **견고성**: 표준 마크업, 유효한 HTML/CSS, 보조기술 호환성

### 나. 웹 접근성 평가 및 점검
구축 완료 후 웹 접근성 평가도구(웹접근성연구소 K-WAH 4.0)로 점검하여 평균 점수가 95점 이상이어야 하며 해당 '웹 접근성 평가 보고서'를 제출하여야 한다.
- **자동 평가**: K-WAH 4.0 도구를 통한 자동 접근성 검사
- **수동 평가**: 전문가에 의한 수동 접근성 검사
- **사용자 테스트**: 실제 장애인 사용자 테스트 수행
- **보고서 작성**: 평가 결과 및 개선사항 상세 보고서 제출

### 다. 웹 표준 준수
전자정부 웹 표준 준수지침 및 W3C(World Wide Web Consortium)의 웹 표준을 준수하여 개발하여야 한다.
- **HTML5 표준**: 시맨틱 마크업, 웹 폼, 멀티미디어 요소 표준 준수
- **CSS3 표준**: 플렉스박스, 그리드, 애니메이션 등 최신 CSS 기능 활용
- **JavaScript ES6+**: 모던 JavaScript 표준 및 모듈 시스템 적용
- **웹 성능**: Core Web Vitals 기준 준수 (LCP < 2.5s, FID < 100ms, CLS < 0.1)

### 라. 다양한 인터넷 환경 접근성
모든 페이지는 다양한 인터넷 환경에서 어려움 없이 접근 가능하여야 한다.

#### (1) 멀티 디바이스 지원
데스크톱PC, 태블릿PC, 패드, 스마트폰 등 모든 인터넷 단말기에서 동일한 서비스를 제공하여야 한다.
- **반응형 디자인**: 320px ~ 1920px 모든 화면 크기 지원
- **터치 최적화**: 모바일 터치 인터페이스 최적화
- **성능 최적화**: 모바일 네트워크 환경 고려한 최적화
- **PWA 지원**: 앱과 유사한 사용자 경험 제공

#### (2) 멀티 브라우저 지원
Explorer, Firefox, Chrome, Safari, Opera 등 다양한 인터넷 브라우저에서 버전에 상관없이 동일한 서비스를 제공하여야 한다. 특히 Explorer인 경우 '호환성 보기 설정'에 주소를 추가했을 때도 동일한 서비스를 제공하여야 한다.
- **브라우저 호환성**: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+, IE11+ 지원
- **폴백 지원**: 구형 브라우저를 위한 폴백 기능 제공
- **크로스 브라우저 테스트**: 주요 브라우저 조합 테스트 완료
- **IE 호환성**: Internet Explorer 호환성 보기 설정 대응
