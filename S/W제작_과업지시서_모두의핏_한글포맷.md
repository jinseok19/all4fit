# S/W 제작 과업지시서 - 모두의핏

## 1. 과업 내용

### 1.1 서비스 기획 및 UI/UX 설계

#### 가. 서비스 기획 및 방향성 수립
- **서비스 핵심 가치 정의**: 체육시설 접근성 향상, 사용자 편의성 극대화, 지역사회 건강증진
- **타겟 사용자 분석**: 연령대별, 지역별, 운동 목적별 사용자 페르소나 정의
- **경쟁사 분석**: 기존 체육시설 플랫폼 벤치마킹 및 차별화 전략 수립
- **서비스 로드맵**: 단계별 기능 확장 계획 및 우선순위 설정

#### 나. UX 설계 (User Journey, IA, Wireframe)
- **사용자 여정 분석**: 시설 검색부터 이용 완료까지 전체 사용자 여정 설계
- **정보구조(IA) 설계**: 직관적인 네비게이션 및 콘텐츠 분류 체계 구축
- **와이어프레임 제작**: 주요 화면별 기능 배치 및 사용자 인터랙션 설계
- **사용성 테스트**: 프로토타입 기반 사용자 테스트 및 개선사항 도출

#### 다. UI 디자인 가이드 제작
- **디자인 시스템 구축**: 컬러 팔레트, 타이포그래피, 아이콘, 컴포넌트 가이드
- **반응형 디자인**: 데스크톱, 태블릿, 모바일 최적화된 UI 설계
- **접근성 고려**: 웹 접근성 지침 2.0 준수한 UI 설계
- **브랜드 아이덴티티**: 모두의핏 브랜드 정체성 반영한 시각적 디자인

#### 라. 산출물 정리 및 개발 파트와의 핸드오프 준비
- **디자인 시안**: 피그마 기반 완성된 UI 디자인 파일 (화면 25개 이상)
- **프론트엔드 가이드**: 개발자를 위한 상세 스펙 및 구현 가이드
- **컴포넌트 라이브러리**: 재사용 가능한 UI 컴포넌트 정의
- **개발 협업 도구**: 피그마, 제플린 등 협업 도구 설정 및 권한 관리

### 1.2 체육시설 통합 플랫폼 주요 서비스를 웹 애플리케이션으로 구축
- 시설 검색 및 필터링: 지역, 종목, 시설 유형별 고급 검색 기능
- 시설 상세 정보 제공: 위치, 연락처, 이용시간, 요금, 시설 사진 등 종합 정보
- 바우처 정보 관리: 스포츠강좌이용권 신청 및 안내 서비스
- 사용자 맞춤 서비스: 즐겨찾기, 검색 기록, 개인화된 추천 기능

### 1.2 실시간 알림서비스 기능 구현 및 모바일 홍보서비스 개발
- 웹 푸시 알림: 브라우저 기반 푸시 알림 서비스 (Service Worker 활용)
- 이벤트 알림: 시설별 이벤트, 공지사항 실시간 알림
- 관리자 알림 시스템: 관리자 대시보드를 통한 푸시 메시지 전송 기능
- SNS 연동: 체육시설 정보 공유 및 홍보 기능 (향후 확장)

### 1.3 웹 애플리케이션 운영 및 컨텐츠 등록 등 관리시스템 구축
- 관리자 대시보드: 직관적인 웹 기반 관리 인터페이스
- 시설 정보 CRUD: 시설 등록, 수정, 삭제, 조회 기능
- 컨텐츠 관리: 이미지, 동영상, 문서 등 멀티미디어 컨텐츠 관리
- 사용자 관리: 회원 가입, 로그인, 권한 관리 시스템
- 통계 및 분석: 접속 통계, 사용자 행동 분석, 성능 모니터링

### 1.4 보안 및 개인정보보호 준수
- 사용자 및 관리자 권한 설정을 통한 접근 제한 기능 구현
- 개인정보보호 강화를 위한 네트워크 구간 보안통신(HTTPS) 적용
- Secure Coding 가이드라인을 준수하여 소스코드 작성

### 1.5 S/W 도입
- 보안통신을 위한 SSL(Secure Socket Layer) 인증서
- 입력 값 노출 및 변조 방지를 위한 클라이언트 사이드 보안
- 인증 및 보안을 위한 PKI 솔루션

## 3. 산출물 ※ 정량적 수치 표현

### 1. 시스템 산출물

#### 가. 서비스 기획 및 UI/UX 설계 산출물 1식
- **서비스 기획서**: 핵심 가치, 타겟 사용자, 경쟁사 분석, 서비스 로드맵 (문서 1개, 30페이지)
- **사용자 여정 맵**: 전체 사용자 여정 분석 및 개선 포인트 도출 (다이어그램 5개)
- **정보구조(IA) 설계서**: 사이트맵, 네비게이션 구조, 콘텐츠 분류 체계 (도면 8개)
- **와이어프레임**: 주요 화면별 기능 배치 및 인터랙션 설계 (화면 25개)
- **UI 디자인 시스템**: 컬러, 타이포그래피, 아이콘, 컴포넌트 가이드 (가이드 1개, 50페이지)
- **피그마 디자인 파일**: 완성된 UI 디자인 및 프로토타입 (파일 1개, 화면 25개 이상)
- **사용성 테스트 보고서**: 프로토타입 테스트 결과 및 개선사항 (보고서 1개, 15페이지)
- **프론트엔드 개발 가이드**: 개발자를 위한 상세 스펙 및 구현 가이드 (가이드 1개, 40페이지)

#### 나. 웹 플랫폼 1식
- 체육시설 통합 플랫폼 웹사이트: https://all4fit.co.kr
- 반응형 웹 애플리케이션: 데스크톱, 태블릿, 모바일 최적화
- PWA(Progressive Web App): 앱과 유사한 사용자 경험 제공
- 도메인 등록 및 SSL 인증서: 보안 통신 환경 구축

#### 나. 웹 플랫폼 소프트웨어 1식 (전산자료 포함)
- 소스코드 패키지: HTML, CSS, JavaScript 전체 소스코드 (5,000라인 이상)
- 관리자 대시보드: 시설 정보 관리 시스템 (화면 10개, 기능 25개)
- 데이터베이스: JSON 기반 시설 정보 데이터 (파일 15개, 레코드 1,000개 이상)
- 배포 스크립트: 자동 배포 및 관리 도구 (스크립트 8개)
- 문서화: API 명세서, 기술 문서, 사용자 매뉴얼 (문서 12개, 총 200페이지)

#### 다. 보안 솔루션 1식
- SSL/TLS 암호화: HTTPS 보안 통신 (인증서 3개 도메인)
- 보안 헤더: CSP, X-Frame-Options 등 보안 설정 (헤더 8개)
- 인증 시스템: JWT 기반 사용자 인증 (토큰 만료시간 24시간)
- 방화벽 설정: UFW 기반 서버 보안 (규칙 15개)

#### 라. 서버 인프라 1식
- AWS EC2 인스턴스: Ubuntu 20.04 LTS 서버 (1대, 8GB RAM)
- Nginx 웹서버: 고성능 웹서버 설정 (설정파일 4개)
- PM2 프로세스 관리: 무중단 배포 시스템 (프로세스 2개)
- 모니터링 도구: 시스템 상태 모니터링 (로그 파일 10개)

#### 마. 웹사이트 등록 (등록비 포함)
- 도메인 등록: all4fit.co.kr 도메인 1년 등록 (1개)
- SSL 인증서: Let's Encrypt 무료 SSL 인증서 (3개 도메인)
- 서버 호스팅: AWS EC2 인스턴스 1년 운영 (1대)
- DNS 설정: 도메인 네임서버 설정 (레코드 8개)

- 보안 설계: 보안 정책 및 암호화 방안 (정책 문서 5개)
- 배포 설계: 서버 구성 및 배포 프로세스 (설정파일 10개)
- 유지보수 가이드: 운영 및 유지보수 매뉴얼 (가이드 8개)
- **총 문서 수: 58개, 총 페이지 수: 150페이지 이상**

**※ 소프트웨어 설계서 제출 방법**

- AWS 계정 ID, 비밀번호, 접근 권한 설정 정보 제공

- 소스코드, 설정파일, 문서 등 모든 기술 자산 클라우드 저장 (용량 2GB 이상)
- USB/CD 물리적 백업과 함께 클라우드 접근 방법 안내

## 2. 세부 사항

### 1. 웹 접근성 및 웹 표준 준수

#### 가. 웹 접근성 지침 준수
모든 페이지는 '한국형 웹 콘텐츠 접근성 지침 2.0'을 준수하여 구축한다.
- 인식의 용이성: 대체 텍스트, 색상 대비, 텍스트 크기 조절 기능 제공
- 운용의 용이성: 키보드 접근성, 충분한 링크 영역, 일관된 네비게이션
- 이해의 용이성: 명확한 제목 구조, 읽기 쉬운 콘텐츠, 오류 메시지 제공
- 견고성: 표준 마크업, 유효한 HTML/CSS, 보조기술 호환성
